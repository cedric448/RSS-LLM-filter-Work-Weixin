# 部署指南

## 1. 环境要求

### 1.1 系统要求

| 组件 | 要求 | 说明 |
|------|------|------|
| 操作系统 | Linux (推荐 Ubuntu 20.04+) | - |
| Python | 3.11+ | 核心运行时 |
| Docker | 20.10+ | 容器运行 |
| Docker Compose | 2.0+ | 服务编排 |
| 磁盘空间 | 10GB+ | 日志和数据存储 |
| 内存 | 2GB+ | n8n + PostgreSQL |

### 1.2 依赖服务

- **今天看啥 VIP**: VIP-3 或 VIP-6 套餐
- **大模型 API**: kimi-k2.5-ioa 访问权限
- **企业微信机器人**: Webhook Key

### 5.2 健康检查脚本

创建 `scripts/health_check.sh`:

```bash
#!/bin/bash

LOG_FILE="/opt/rss-ai-push/logs/auto_push.log"
ALERT_EMAIL="admin@example.com"

# 检查最近一次运行时间
LAST_RUN=$(grep "🚀 RSS 智能筛选自动推送" "$LOG_FILE" | tail -1 | awk '{print $4, $5}')
LAST_RUN_TIME=$(date -d "$LAST_RUN" +%s 2>/dev/null || echo 0)
CURRENT_TIME=$(date +%s)
TIME_DIFF=$((CURRENT_TIME - LAST_RUN_TIME))

# 如果超过 4 小时未运行，发送告警
if [ $TIME_DIFF -gt 14400 ]; then
    echo "警告: RSS 推送系统超过 4 小时未运行" | mail -s "RSS 推送告警" $ALERT_EMAIL
fi

# 检查错误率
ERROR_COUNT=$(grep "✗" "$LOG_FILE" | tail -100 | wc -l)
if [ $ERROR_COUNT -gt 20 ]; then
    echo "警告: 最近100行日志中发现 $ERROR_COUNT 个错误" | mail -s "RSS 推送高错误率" $ALERT_EMAIL
fi
```

配置定时健康检查:

```bash
# 每小时检查一次
0 * * * * /opt/rss-ai-push/scripts/health_check.sh
```

### 5.3 性能监控

创建 `scripts/stats.py`:

```python
#!/usr/bin/env python3
import sqlite3
from datetime import datetime, timedelta

DB_PATH = "/opt/rss-ai-push/data/push_history.db"

conn = sqlite3.connect(DB_PATH)
cursor = conn.cursor()

# 统计今天推送数
today = datetime.now().strftime('%Y-%m-%d')
cursor.execute("SELECT COUNT(*) FROM push_history WHERE DATE(push_time) = ?", (today,))
today_count = cursor.fetchone()[0]

# 统计最近7天推送数
week_ago = (datetime.now() - timedelta(days=7)).strftime('%Y-%m-%d')
cursor.execute("SELECT COUNT(*) FROM push_history WHERE DATE(push_time) >= ?", (week_ago,))
week_count = cursor.fetchone()[0]

# 按来源统计
cursor.execute("""
    SELECT source_name, COUNT(*) as count
    FROM push_history
    WHERE DATE(push_time) >= ?
    GROUP BY source_name
    ORDER BY count DESC
    LIMIT 10
""", (week_ago,))
top_sources = cursor.fetchall()

print(f"今日推送: {today_count} 篇")
print(f"近7天推送: {week_count} 篇")
print(f"日均推送: {week_count / 7:.1f} 篇")
print("\n最近7天 Top 10 来源:")
for source, count in top_sources:
    print(f"  {source}: {count} 篇")

conn.close()
```

## 6. 备份与恢复

### 6.1 数据备份

```bash
#!/bin/bash
# scripts/backup.sh

BACKUP_DIR="/opt/rss-ai-push/backups"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"

# 备份数据库
cp /opt/rss-ai-push/data/push_history.db "$BACKUP_DIR/push_history_$DATE.db"

# 备份配置文件
tar -czf "$BACKUP_DIR/config_$DATE.tar.gz" -C /opt/rss-ai-push config/

# 清理30天前的备份
find "$BACKUP_DIR" -name "*.db" -mtime +30 -delete
find "$BACKUP_DIR" -name "*.tar.gz" -mtime +30 -delete

echo "备份完成: $DATE"
```

配置每日备份:

```bash
# 每天凌晨 1 点备份
0 1 * * * /opt/rss-ai-push/scripts/backup.sh
```

### 6.2 数据恢复

```bash
# 恢复数据库
cp /opt/rss-ai-push/backups/push_history_20260223_010000.db /opt/rss-ai-push/data/push_history.db

# 恢复配置
tar -xzf /opt/rss-ai-push/backups/config_20260223_010000.tar.gz -C /opt/rss-ai-push/
```

## 7. 故障排查

### 7.1 常见问题

#### 问题 1: Crontab 未执行

**排查步骤**:

```bash
# 1. 检查 crontab 是否配置
crontab -l

# 2. 检查 cron 服务状态
systemctl status cron

# 3. 查看 cron 日志
grep CRON /var/log/syslog | tail -20

# 4. 手动执行测试
cd /root/project-wb/n8n && python3 src/auto_push.py
```

#### 问题 2: 推送失败

**排查步骤**:

```bash
# 1. 查看日志
tail -50 logs/auto_push.log | grep "✗"

# 2. 测试网络连接
curl -I http://www.jintiankansha.me
curl -I http://your-llm-api-host
curl -I https://qyapi.weixin.qq.com

# 3. 验证 API Key
curl -X POST "http://your-llm-api-host/agent" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"model":"kimi-k2.5-ioa","messages":[{"role":"user","content":"test"}]}'
```

#### 问题 3: 数据库锁定

**解决方法**:

```bash
# 检查数据库文件权限
ls -lh data/push_history.db

# 如果锁定，删除锁文件
rm -f data/push_history.db-journal

# 重建数据库 (慎用,会丢失历史记录)
rm data/push_history.db
python3 src/auto_push.py
```

### 7.2 性能问题

#### 执行时间过长

**优化建议**:

1. 减少 RSS 源数量
2. 降低 `MAX_ARTICLES_PER_SOURCE` (默认 20)
3. 增加 `MAX_CONCURRENT_RSS` (默认 3)
4. 检查网络延迟

```python
# 修改 src/auto_push.py
MAX_ARTICLES_PER_SOURCE = 10  # 从 20 降到 10
MAX_CONCURRENT_RSS = 5        # 从 3 增到 5
```

#### 限流频繁

**优化建议**:

```python
# 增加请求间隔
REQUEST_INTERVAL = 2.0  # 从 1.0 增到 2.0

# 降低并发数
MAX_CONCURRENT_RSS = 2  # 从 3 降到 2
```

## 系统概述

### 系统架构

```
┌─────────────────────────────────────────┐
│         Cron 定时任务                    │
│    (9:00, 13:00, 20:00)                 │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│      Python 自动推送脚本                 │
│   /root/project-wb/n8n/src/auto_push.py│
└──────────────┬──────────────────────────┘
               │
               ├──▶ RSS源 (今天看啥API)
               ├──▶ AI模型API
               ├──▶ SQLite数据库 (去重)
               └──▶ 企业微信Webhook
```

### 系统组件

| 组件 | 路径 | 说明 |
|------|------|------|
| 自动推送脚本 | `/root/project-wb/n8n/src/auto_push.py` | 核心业务逻辑 |

## 监控与告警

### 监控指标

#### 1. 推送成功率

**脚本**: `scripts/check_push_success_rate.sh`

```bash
cat > /root/project-wb/n8n/scripts/check_push_success_rate.sh << 'EOF'
#!/bin/bash

LOG_FILE="/root/project-wb/n8n/logs/auto_push.log"
TODAY=$(date +%Y-%m-%d)

# 统计今天的推送次数
TOTAL=$(grep "$TODAY.*推送完成" "$LOG_FILE" | wc -l)
SUCCESS=$(grep "$TODAY.*✅ 推送成功" "$LOG_FILE" | wc -l)

if [ $TOTAL -eq 0 ]; then
    echo "今天尚未执行推送"
    exit 0
fi

SUCCESS_RATE=$((SUCCESS * 100 / TOTAL))

echo "今天推送统计:"
echo "  总次数: $TOTAL"
echo "  成功次数: $SUCCESS"
echo "  成功率: $SUCCESS_RATE%"

# 告警阈值: 成功率 < 80%
if [ $SUCCESS_RATE -lt 80 ]; then
    echo "⚠️ 警告: 推送成功率过低!"
    exit 1
fi
EOF

chmod +x /root/project-wb/n8n/scripts/check_push_success_rate.sh
```

#### 2. RSS 源健康检查

**脚本**: `scripts/check_rss_health.sh`

```bash
cat > /root/project-wb/n8n/scripts/check_rss_health.sh << 'EOF'
#!/bin/bash

CONFIG_FILE="/root/project-wb/n8n/config/rss-sources.json"
FAILED_SOURCES=()

echo "检查 RSS 源健康状态..."
echo

# 使用 Python 解析 JSON 并测试每个 RSS 源
python3 << END
import json
import requests

with open('$CONFIG_FILE', 'r', encoding='utf-8') as f:
    config = json.load(f)

failed_count = 0
for source in config['sources']:
    if not source.get('enabled', False):
        continue
    
    name = source['name']
    slug = source['slug']
    url = f"http://www.jintiankansha.me/api3/query/get_topics_by_one_column"
    params = {
        'user': '13167171761@163.com',
        'token': 'IXhXzdNDnO',
        'slug': slug,
        'count': 1
    }
    
    try:
        response = requests.get(url, params=params, timeout=10)
        if response.status_code == 200:
            print(f"✅ {name}: 正常")
        else:
            print(f"❌ {name}: HTTP {response.status_code}")
            failed_count += 1
    except Exception as e:
        print(f"❌ {name}: {str(e)}")
        failed_count += 1

if failed_count > 0:
    print(f"\n⚠️ 警告: {failed_count} 个RSS源异常")
    exit(1)
else:
    print(f"\n✅ 所有RSS源健康")
    exit(0)
END
EOF

chmod +x /root/project-wb/n8n/scripts/check_rss_health.sh
```

## 故障排查

### 常见问题诊断流程

```
故障现象
  │
  ├─▶ 未收到推送消息
  │   ├─ 检查 cron 是否执行
  │   ├─ 查看日志错误信息
  │   ├─ 手动执行脚本测试
  │   └─ 检查企业微信 Webhook
  │
  ├─▶ 推送内容错误/缺失
  │   ├─ 检查 RSS 源可访问性
  │   ├─ 检查 AI 筛选结果
  │   └─ 验证消息格式化逻辑
  │
  ├─▶ 重复推送相同文章
  │   ├─ 检查数据库去重逻辑
  │   ├─ 验证文章 URL 唯一性
  │   └─ 检查数据库记录
  │
  └─▶ 脚本执行缓慢
      ├─ 检查网络连接
      ├─ 查看 API 响应时间
      └─ 优化并发和批处理
```

## 去重机制说明

### 去重架构

系统使用基于哈希的去重机制,确保已推送的文章不会重复推送。

**核心流程**:

```
RSS文章获取
    ↓
生成文章哈希 (MD5)
    ↓
查询数据库 (is_article_pushed)
    ↓
过滤已推送文章 (filter_new_articles)
    ↓
AI筛选处理
    ↓
推送到企业微信
    ↓
标记为已推送 (mark_articles_as_pushed)
    ↓
写入SQLite数据库
```

### 哈希生成策略

**优先级**:
1. **优先使用链接**: 文章链接是最可靠的唯一标识
2. **降级使用标题+来源**: 当文章没有链接时使用

**代码实现** (`auto_push.py`):

```python
def get_article_hash(article):
    """生成文章唯一哈希（基于链接或标题）"""
    # 优先使用链接，如果没有链接则使用标题+来源
    unique_str = article.get('link', '') or f"{article.get('title', '')}_{article.get('source_name', '')}"
    return hashlib.md5(unique_str.encode('utf-8')).hexdigest()
```

### 数据库设计

**表结构**:

```sql
CREATE TABLE IF NOT EXISTS push_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    article_hash TEXT UNIQUE NOT NULL,  -- 唯一约束防止重复
    title TEXT,
    link TEXT,
    source_name TEXT,
    push_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引优化查询性能
CREATE INDEX IF NOT EXISTS idx_article_hash ON push_history(article_hash);
CREATE INDEX IF NOT EXISTS idx_push_time ON push_history(push_time);
```

**关键特性**:
- ✅ `article_hash UNIQUE NOT NULL` 唯一约束防止重复插入
- ✅ `idx_article_hash` 索引加速查询(<1ms/篇)
- ✅ `push_time` 时间戳记录推送时间

### 去重核心函数

#### 1. 查询已推送

```python
def is_article_pushed(article):
    """检查文章是否已推送"""
    article_hash = get_article_hash(article)
    
    conn = sqlite3.connect(PUSH_HISTORY_DB)
    cursor = conn.cursor()
    
    cursor.execute(
        'SELECT COUNT(*) FROM push_history WHERE article_hash = ?',
        (article_hash,)
    )
    count = cursor.fetchone()[0]
    
    conn.close()
    return count > 0
```

#### 2. 过滤新文章

```python
def filter_new_articles(articles):
    """过滤出未推送的新文章"""
    new_articles = []
    duplicate_count = 0
    
    for article in articles:
        if not is_article_pushed(article):
            new_articles.append(article)
        else:
            duplicate_count += 1
    
    if duplicate_count > 0:
        print(f"  ℹ️ 过滤掉 {duplicate_count} 篇已推送的文章")
    
    return new_articles
```

#### 3. 标记已推送

```python
def mark_articles_as_pushed(articles):
    """标记文章为已推送"""
    if not articles:
        return
    
    conn = sqlite3.connect(PUSH_HISTORY_DB)
    cursor = conn.cursor()
    
    for article in articles:
        article_hash = get_article_hash(article)
        
        try:
            cursor.execute('''
                INSERT INTO push_history (article_hash, title, link, source_name)
                VALUES (?, ?, ?, ?)
            ''', (
                article_hash,
                article.get('title', '')[:200],
                article.get('link', ''),
                article.get('source_name', '')
            ))
        except sqlite3.IntegrityError:
            # 唯一约束冲突，跳过（已存在）
            pass
    
    conn.commit()
    conn.close()
```

### 边缘场景处理

#### 场景 1: 相同文章但标题修改

**情况**: 文章链接不变,编辑修改了标题
**结果**: ✅ 被识别为同一篇文章(基于链接)
**行为**: 不会重复推送

#### 场景 2: 相同内容但链接变化

**情况**: 文章迁移到新URL,标题相同
**结果**: ❌ 被识别为不同文章
**行为**: 会重复推送
**说明**: 这是合理的,链接变化通常意味着内容更新

#### 场景 3: 无链接文章的标题相同

**情况**: 两篇都没有链接,标题和来源相同
**结果**: ✅ 被识别为同一篇文章
**行为**: 不会重复推送

#### 场景 4: 不同来源的相同标题

**情况**: 标题相同但来自不同RSS源
**结果**: ❌ 被识别为不同文章
**行为**: 会推送两次
**说明**: 这是合理的,不同来源应独立处理

```python
def clean_old_history():
    """清理30天前的推送历史"""
    cutoff_date = datetime.now() - timedelta(days=30)
    
    conn = sqlite3.connect(PUSH_HISTORY_DB)
    cursor = conn.cursor()
    
    cursor.execute(
        'DELETE FROM push_history WHERE push_time < ?',
        (cutoff_date,)
    )
    
    deleted = cursor.rowcount
    conn.commit()
    conn.close()
    
    return deleted
```

- ✅ RSS文章更新周期通常<7天

### 维护窗口

**建议维护时间**: 每周日凌晨 2:00-4:00

**维护内容**:
1. 系统更新检查
2. 日志清理
3. 数据库优化
4. 性能分析
5. 备份验证

**维护脚本**: `scripts/weekly_maintenance.sh`

```bash
cat > /root/project-wb/n8n/scripts/weekly_maintenance.sh << 'EOF'
#!/bin/bash

echo "=========================================="
echo "RSS 系统周维护 - $(date)"
echo "=========================================="

# 1. 完整备份
echo "[1] 执行完整备份..."
/root/project-wb/n8n/scripts/backup_full.sh

# 2. 数据库优化
echo "[2] 优化数据库..."
sqlite3 /root/project-wb/n8n/data/push_history.db "VACUUM;"
echo "数据库优化完成"

# 3. 清理过期日志
echo "[3] 清理过期日志..."
find /root/project-wb/n8n/logs -name "*.log.*" -mtime +30 -delete
echo "日志清理完成"

# 4. 清理过期备份
echo "[4] 清理过期备份..."
find /root/backups/rss-system -name "*.tar.gz" -mtime +30 -delete
find /root/backups/rss-system -name "*.db.gz" -mtime +30 -delete
echo "备份清理完成"

# 5. 系统健康检查
echo "[5] 系统健康检查..."
/root/check_rss_system.sh

echo "=========================================="
echo "周维护完成"
echo "=========================================="
EOF

chmod +x /root/project-wb/n8n/scripts/weekly_maintenance.sh

# 添加到 crontab (每周日凌晨2点)

```

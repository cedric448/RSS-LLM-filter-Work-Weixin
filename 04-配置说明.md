## 2. config/rss-sources.json

### 2.1 配置格式

```json
{
  "sources": [
    {
      "name": "AI前线",
      "slug": "ai-frontiers",
      "url": "https://www.jintiankansha.me/column/ai-frontiers",
      "enabled": true
    }
  ]
}
```

### 2.2 字段说明

| 字段 | 类型 | 必需 | 说明 | 示例 |
|------|------|------|------|------|
| `name` | string | ✅ | RSS 源名称,显示在推送消息中 | "AI前线" |
| `slug` | string | ✅ | 今天看啥的公众号 slug,用于 API4 获取文章 | "ai-frontiers" |
| `url` | string | ❌ | 今天看啥专栏页面链接 (仅供参考) | "https://..." |
| `enabled` | boolean | ✅ | 是否启用该源 | true/false |

### 2.3 获取 slug 的方法

#### 方法 1: 从今天看啥网站获取

1. 登录 https://www.jintiankansha.me
2. 进入专栏页面
3. URL 中 `/column/{slug}` 即为 slug

**示例**:
```
URL: https://www.jintiankansha.me/column/ai-frontiers
Slug: ai-frontiers
```

#### 方法 2: 使用测试脚本

```bash
# 测试单个 slug
python3 << EOF
import requests

API4_USER = "your_email@example.com"
API4_TOKEN = "your_token"
slug = "ai-frontiers"

url = f"http://www.jintiankansha.me/api3/query/get_topics_by_one_column"
params = {"user": API4_USER, "token": API4_TOKEN, "slug": slug}

response = requests.get(url, params=params)
print(f"状态码: {response.status_code}")
print(f"响应: {response.json()}")
EOF
```

### 2.5 注意事项

1. **slug 必须正确**: 错误的 slug 会导致 API 返回 404
2. **enabled 控制**: 临时禁用某个源时,将 `enabled` 设为 `false`
3. **源数量限制**: 建议 < 200 个源 (今天看啥 VIP-6 套餐限制)
4. **名称唯一性**: 建议不同源使用不同名称,避免混淆

### 3.1 配置格式

```json
{
  "categories": [
    {
      "name": "AIGC",
      "keywords": ["生成式AI", "AIGC", "内容生成", "文生图", "文生视频"],

  "threshold": 0.7
}
```

### 3.2 字段说明

#### categories (数组)

| 字段 | 类型 | 必需 | 说明 | 示例 |
|------|------|------|------|------|
| `name` | string | ✅ | 分类名称,显示在推送消息中 | "AIGC" |
| `keywords` | array | ✅ | 关键词列表,用于语义匹配 | ["生成式AI", "AIGC"] |
| `description` | string | ❌ | 分类描述,用于构建 AI 提示词 | "生成式人工智能内容" |

#### 全局配置

| 字段 | 类型 | 必需 | 说明 | 默认值 |
|------|------|------|------|--------|
| `match_mode` | string | ❌ | 匹配模式: `semantic`(语义) 或 `keyword`(精确) | "semantic" |
| `threshold` | float | ❌ | 置信度阈值 (0.0-1.0) | 0.7 |

### 3.4 配置建议

#### 3.4.1 关键词设计原则

1. **语义丰富**: 包含同义词、相关词、英文缩写
2. **覆盖全面**: 考虑不同表达方式
3. **避免过泛**: 不要使用"技术"、"新闻"等通用词

**好的示例**:
```json
["生成式AI", "AIGC", "内容生成", "文生图", "Midjourney"]
```

**不好的示例**:
```json
["AI", "技术", "创新"]  // 太泛
```

#### 3.4.2 置信度阈值调整

| 阈值 | 效果 | 适用场景 |
|------|------|---------|
| 0.5 | 宽松,更多文章通过 | 初期测试,广撒网 |
| 0.7 | 推荐,平衡精准度和召回率 | 生产环境 |
| 0.9 | 严格,只推送高度相关文章 | 精准推送 |

**调整示例**:
```json
{
  "threshold": 0.8  // 提高到 0.8,更严格筛选
}
```

#### 3.4.3 分类数量建议

- **最少**: 3-5 个核心分类
- **推荐**: 5-10 个分类
- **最多**: 不超过 15 个 (过多会影响 AI 判断精度)

### 5.3 配置验证

创建 `scripts/validate_config.py`:

```python
#!/usr/bin/env python3
import json
import sys

def validate_rss_sources(config_path):
    """验证 RSS 源配置"""
    try:
        with open(config_path, 'r') as f:
            data = json.load(f)
        
        sources = data.get('sources', [])
        if not sources:
            print("❌ 错误: 没有配置任何 RSS 源")
            return False
        
        for i, source in enumerate(sources):
            if not source.get('name'):
                print(f"❌ 错误: 第 {i+1} 个源缺少 name 字段")
                return False
            if not source.get('slug'):
                print(f"❌ 错误: 源 '{source['name']}' 缺少 slug 字段")
                return False
            if 'enabled' not in source:
                print(f"⚠️  警告: 源 '{source['name']}' 缺少 enabled 字段,默认为 false")
        
        enabled_count = sum(1 for s in sources if s.get('enabled', False))
        print(f"✅ RSS 源配置有效: 共 {len(sources)} 个源, {enabled_count} 个已启用")
        return True
        
    except json.JSONDecodeError as e:
        print(f"❌ 错误: JSON 格式无效 - {e}")
        return False
    except Exception as e:
        print(f"❌ 错误: {e}")
        return False

def validate_keywords(config_path):
    """验证关键词配置"""
    try:
        with open(config_path, 'r') as f:
            data = json.load(f)
        
        categories = data.get('categories', [])
        if not categories:
            print("❌ 错误: 没有配置任何分类")
            return False
        
        for cat in categories:
            if not cat.get('name'):
                print(f"❌ 错误: 分类缺少 name 字段")
                return False
            if not cat.get('keywords') or len(cat['keywords']) == 0:
                print(f"❌ 错误: 分类 '{cat['name']}' 没有关键词")
                return False
        
        threshold = data.get('threshold', 0.7)
        if not (0.0 <= threshold <= 1.0):
            print(f"⚠️  警告: 置信度阈值 {threshold} 超出范围 [0.0, 1.0]")
        
        print(f"✅ 关键词配置有效: 共 {len(categories)} 个分类, 阈值 {threshold}")
        return True
        
    except Exception as e:
        print(f"❌ 错误: {e}")
        return False

if __name__ == '__main__':
    rss_ok = validate_rss_sources('/root/project-wb/n8n/config/rss-sources.json')
    keywords_ok = validate_keywords('/root/project-wb/n8n/config/keywords.json')
    
    if rss_ok and keywords_ok:
        print("\n✅ 所有配置验证通过")
        sys.exit(0)
    else:
        print("\n❌ 配置验证失败")
        sys.exit(1)
```

运行验证:
```bash
python3 scripts/validate_config.py
```

## 6. 常见配置问题

### 6.1 问题: 无法获取文章

**可能原因**:
- slug 错误
- API 认证失败
- 网络连接问题

**解决方法**:
```bash
# 测试单个 slug
curl -X GET "http://www.jintiankansha.me/api3/query/get_topics_by_one_column" \
  -G \
  --data-urlencode "user=YOUR_EMAIL" \
  --data-urlencode "token=YOUR_TOKEN" \
  --data-urlencode "slug=ai-frontiers"
```

### 6.2 问题: AI 筛选不准确

**可能原因**:
- 关键词配置不合理
- 置信度阈值过低/过高

**解决方法**:
1. 查看筛选日志,分析误判文章
2. 调整关键词配置
3. 调整置信度阈值

### 6.3 问题: 推送消息格式错误

**可能原因**:
- Markdown 格式错误
- 特殊字符未转义

**解决方法**:
检查 `src/wechat_pusher.py` 的格式化函数。

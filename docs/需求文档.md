## 项目概述

### 项目目标

通过**今天看啥RSS服务**订阅公众号内容，使用**大语言模型**进行智能语义筛选，将符合AI主题的优质文章自动推送到**企业微信群**，帮助团队及时获取AI领域的最新资讯。

### 应用场景

- AI团队日常资讯跟踪
- 技术趋势监测
- 竞品动态分析
- 投融资情报收集

### 核心价值

1. **智能筛选**: 使用大模型语义理解，避免关键词匹配的误报和漏报
2. **自动化运行**: 定时推送，无需人工干预
3. **去重机制**: 确保每篇文章只推送一次
4. **灵活配置**: 支持动态调整RSS源、筛选规则、推送时间

### FR-01 RSS订阅管理

**需求描述**: 支持订阅和管理多个公众号的RSS源

**详细要求**:
- 支持订阅200个以内的公众号（今天看啥VIP-6套餐限制）

### FR-02 文章获取与解析

**需求描述**: 定期从RSS源获取最新文章并解析

**详细要求**:
- 每个RSS源获取最近20篇文章

- 支持并发获取多个RSS源（最大并发5个）

**验收标准**:

- ✅ 正确处理网络异常和RSS格式异常

### FR-03 AI智能筛选

**需求描述**: 使用大语言模型进行内容相关性判断

**详细要求**:

**筛选维度**:

### FR-04 去重机制

**需求描述**: 确保每篇文章只推送一次

**详细要求**:
- 使用文章URL作为唯一标识
- 维护推送历史数据库（SQLite）
- 推送前检查文章是否已推送
- 历史记录保留30天，自动清理过期数据

**去重逻辑**:
```
1. 获取文章列表
2. 查询数据库，过滤已推送的URL
3. 仅对未推送的文章进行AI筛选
4. 推送成功后，记录URL和推送时间到数据库
```

**验收标准**:
- ✅ 相同文章不会重复推送
- ✅ 去重查询性能 < 100ms
- ✅ 数据库自动清理过期记录

### 5.3 去重模块

**模块**: `src/auto_push.py::mark_articles_as_pushed()`

**数据库表结构**:
```sql
CREATE TABLE push_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    url TEXT NOT NULL UNIQUE,
    title TEXT,
    pushed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    source TEXT
);

CREATE INDEX idx_url ON push_history(url);
CREATE INDEX idx_pushed_at ON push_history(pushed_at);
```

**去重逻辑**:
```python
def filter_unpushed_articles(articles):
    """
    过滤掉已推送的文章
    """
    db = sqlite3.connect('data/push_history.db')
    cursor = db.cursor()
    
    unpushed = []
    for article in articles:
        url = article['url']
        cursor.execute('SELECT 1 FROM push_history WHERE url = ?', (url,))
        if not cursor.fetchone():
            unpushed.append(article)
    
    db.close()
    return unpushed

def mark_articles_as_pushed(articles):
    """
    标记文章为已推送
    """
    db = sqlite3.connect('data/push_history.db')
    cursor = db.cursor()
    
    for article in articles:
        cursor.execute('''
            INSERT OR IGNORE INTO push_history (url, title, source)
            VALUES (?, ?, ?)
        ''', (article['url'], article['title'], article['source']))
    
    db.commit()
    db.close()
```

```python
def format_message(articles, total_count):
    """
    格式化为Markdown消息
    """
    now = datetime.now()
    timestamp = now.strftime('%Y-%m-%d %H:%M')
    
    # 消息头部
    message = f"## 📰 AI资讯精选 ({timestamp})\n\n"
    message += f"本期筛选 {total_count} 篇文章，推送 {len(articles)} 篇\n\n"
    message += "---\n\n"
    
    # 文章列表
    for i, article in enumerate(articles, 1):
        message += f"### {i}. {article['title']}\n"
        message += f"**分类**: {' | '.join(article['categories'])}  \n"
        message += f"**来源**: {article['source']}  \n"
        
        # 摘要（截取前100字）
        summary = article.get('summary', '')
        if len(summary) > 100:
            summary = summary[:100] + '...'
        message += f"**摘要**: {summary}  \n"
        
        message += f"[阅读原文]({article['url']})\n\n"
        message += "---\n\n"
    
    # 消息尾部
    message += "*Powered by RSS智能筛选系统*"
    
    # 检查长度限制（企业微信限制4096字符）
    if len(message) > 4096:
        message = message[:4090] + "...\n\n*内容过长已截断*"
    
    return message
```

```python
def push_to_wechat(message):
    """
    推送消息到企业微信
    """
    webhook_url = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send"
    webhook_url += "?key=defd0b35-256b-40ba-a513-c21feb5955a5"
    
    payload = {
        "msgtype": "markdown",
        "markdown": {
            "content": message
        }
    }
    
    max_retries = 3
    for attempt in range(max_retries):
        try:
            response = requests.post(
                webhook_url,
                json=payload,
                timeout=10
            )
            
            if response.status_code == 200:
                result = response.json()
                if result.get('errcode') == 0:
                    return True, "推送成功"
                else:
                    error_msg = result.get('errmsg', '未知错误')
                    return False, f"推送失败: {error_msg}"
            else:
                return False, f"HTTP错误: {response.status_code}"
                
        except Exception as e:
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)  # 指数退避
                continue
            else:
                return False, f"推送异常: {str(e)}"
```

**响应格式**:

```json
  "errcode": 0,
  "errmsg": "ok"
}
```

### 推送时间安排

|------|-----|---------|---------|

| 20:00 | 晚间 | 下班后 | 深度分析、技术文章 |

### 推送频率控制

**基本策略**:
- 每天推送3次（固定时间）
- 每次推送10-20篇文章
- 推送间隔不少于4小时

**动态调整**:
- 如果某时段无相关文章，则跳过推送
- 如果文章数量过多（>30篇），分两次推送
- 重大新闻可以单独推送（手动触发）

### 推送内容优先级

**排序规则**:
1. **时效性**: 优先推送最新发布的文章
2. **重要性**: 大模型发布 > 融资新闻 > 产品更新
3. **来源权威性**: 官方账号 > 媒体账号 > 个人账号
4. **完整性**: 有摘要 > 无摘要

**实现逻辑**:
```python
def sort_articles(articles):
    """
    对文章进行排序
    """
    def get_priority(article):
        priority = 0
        
        # 分类权重
        category_weight = {
            "大模型": 100,
            "AI融资": 80,
            "AIGC": 70,
            "AI产品": 60,
            "AI初创": 50
        }
        for cat in article['categories']:
            priority += category_weight.get(cat, 0)
        
        # 来源权重
        source_weight = {
            "机器之心": 20,
            "新智元": 20,
            "量子位": 15,
            "创业邦": 10
        }
        priority += source_weight.get(article['source'], 0)
        
        # 时效性（最近的文章加分）
        # publish_time越近，分数越高
        
        return priority
    
    return sorted(articles, key=get_priority, reverse=True)
```

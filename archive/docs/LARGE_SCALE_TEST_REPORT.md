## 4. 瓶颈分析

### 4.1 RSS获取瓶颈

**问题**: 55%的源获取失败/超时

**根本原因**:
1. **RSS服务端限流**: `rss.jintiankansha.me` 对大量并发请求限流
2. **网络不稳定**: 部分源连接不稳定
3. **超时设置**: 15秒超时可能对部分慢速源不够

**改进建议**:
```python
# 方案1: 增加重试机制
def fetch_with_retry(url, source_name, max_retries=2):
    for i in range(max_retries):
        try:
            return fetch_rss_articles(url, source_name)
        except:
            if i < max_retries - 1:
                time.sleep(5)  # 等待5秒重试
            continue
    return []

# 方案2: 动态调整超时
RSS_FETCH_TIMEOUT = 30  # 增加到30秒

# 方案3: 降低并发数
MAX_CONCURRENT_RSS = 5  # 从10降到5
```

### 4.2 AI解析瓶颈

**问题**: 83.3%文章未解析 (75/90)

**根本原因**:
1. **LLM响应格式不一致**: 返回的JSON格式可能不符合预期
2. **文章标题过长或格式复杂**: 导致解析失败
3. **LLM筛选标准不稳定**: 不同批次判断标准不一致

**影响**:
- 虽然未解析率高,但实际筛选出10篇相关文章
- 未解析不影响已解析文章的推送
- 容错处理完善,不会导致程序崩溃
